Loading images with OpenCV

Reading, displaying, and understanding what cv2.imread actually returns

OpenCV's cv2.imread() is the entry point for almost every CV pipeline. It decodes an image file and returns a NumPy array — ready for manipulation without any extra conversion step.
Critical gotcha: OpenCV loads images in BGR order, not RGB. If you pass a cv2 image directly to matplotlib or a neural network expecting RGB, colors will be wrong.

cv2 · imread basics
import cv2
import numpy as np

# Load an image (BGR format by default)
img = cv2.imread("photo.jpg")           # color (BGR)
gray = cv2.imread("photo.jpg", 0)       # grayscale
unchanged = cv2.imread("icon.png", -1) # keep alpha

print(img.shape)   # (height, width, 3)
print(img.dtype)   # uint8

# Convert BGR → RGB for matplotlib / PyTorch
rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)

# Save result
cv2.imwrite("output.jpg", img)
When cv2.imread() fails (file not found, bad path), it silently returns None instead of raising an error. Always check for None before proceeding.

cv2 · resize


import cv2

img = cv2.imread("photo.jpg")

# Exact dimensions
resized = cv2.resize(img, (640, 480))  # (width, height)!

# Scale factor (0.5 = half size)
half = cv2.resize(img, None, fx=0.5, fy=0.5,
                  interpolation=cv2.INTER_AREA)

# Preserve aspect ratio
h, w = img.shape[:2]
target_w = 800
scale = target_w / w
new_h = int(h * scale)
resized_ar = cv2.resize(img, (target_w, new_h),
                         interpolation=cv2.INTER_LINEAR)

        

        
Width vs height order: cv2.resize() takes (width, height) as the destination size — the opposite of NumPy's .shape which returns (height, width, channels). This trips up everyone at first.

Cropping & ROI
NumPy slicing as the native crop operation

        

        
In OpenCV, cropping is just NumPy array slicing — no special function needed. A Region of Interest (ROI) is a sub-array view. The syntax is img[y1:y2, x1:x2], not (x, y, w, h).

cropping


# Crop: img[y1:y2, x1:x2]
roi = img[50:200, 100:350]

# Crop with bbox [x,y,w,h]
x, y, w, h = 100, 50, 250, 150
roi = img[y:y+h, x:x+w]

# Center crop to 224×224
H, W = img.shape[:2]
cy, cx = H//2, W//2
roi = img[cy-112:cy+112,
          cx-112:cx+112]

          

             
 View, not copy

            
NumPy slices are views into the original array. Modifying roi modifies img too. Use roi = img[...].copy() to make an independent copy.

Flipping & rotating
cv2.flip and getRotationMatrix2D for geometric transforms

        

        
Flipping and rotation are the simplest forms of data augmentation — free labels, instant dataset expansion. Understanding the geometry helps you apply them correctly in production pipelines.

cv2.flip


# 0  = vertical flip (top↔bottom)
# 1  = horizontal flip (left↔right)
# -1 = both axes
flipped = cv2.flip(img, 1)

# Rotation around center
h, w = img.shape[:2]
M = cv2.getRotationMatrix2D(
    (w//2, h//2),  # center (cx,cy)
    45,              # angle degrees CCW
    1.0              # scale
)
rot = cv2.warpAffine(img, M, (w, h))

          
data augmentation


import random

def augment(img):
    # 50% chance horizontal flip
    if random.random() > 0.5:
        img = cv2.flip(img, 1)

    # Random rotation ±15°
    angle = random.uniform(-15, 15)
    h, w = img.shape[:2]
    M = cv2.getRotationMatrix2D(
            (w//2, h//2), angle, 1.0)
    img = cv2.warpAffine(img, M, (w, h))
    return img

Bitwise operations
AND, OR, XOR, NOT — pixel-level binary logic

        

        
Bitwise operations work on each bit of each pixel channel independently. They are the core of masking — selectively showing or hiding parts of an image. Mastering them is essential for background removal, image compositing, and ROI selection.

AND: keeps pixels where BOTH images are white (255). Used to apply a binary mask.

          

        


        

          
cv2 · bitwise ops


import cv2
import numpy as np

# Create a binary mask (white circle on black bg)
mask = np.zeros((200, 200), dtype=np.uint8)
cv2.circle(mask, (100, 100), 80, 255, -1)

# AND: keep only the masked region
result = cv2.bitwise_and(img, img, mask=mask)

# NOT: invert the mask
inv_mask = cv2.bitwise_not(mask)

# OR: combine two images
combined = cv2.bitwise_or(img_a, img_b)

# XOR: show differences
diff = cv2.bitwise_xor(img_a, img_b)

Change detection

            
XOR two frames to reveal exactly which pixels changed between them.

          

          

            

            
ROI selection

            
Use NOT to invert a mask and selectively apply different processing to foreground vs background.
