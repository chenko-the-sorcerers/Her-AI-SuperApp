Spatial Convolution

What actually happens when you apply a filter — pixel by pixel math

Every image filter — blur, sharpen, edge detect — boils down to spatial convolution: slide a small weight matrix (the kernel) over every pixel, multiply element-wise with the local neighborhood, sum to get one output value. The kernel defines what the filter does.
Kernel = filter = mask — all three terms mean the same thing. A 3×3 kernel examines 9 pixels at once. Kernel weights summing to 1 = neutral brightness; >1 brightens; <1 darkens.

                    [-1, 5,-1],

# Sobel X (vertical edges)
k_sx = np.array([[-1,0,1],[-2,0,2],[-1,0,1]], dtype=np.float32)
edges_x = cv2.filter2D(img, -1, k_sx)

Gaussian Blur
Noise removal with weighted averaging — nearby pixels matter more

        

        
Box blur weights all neighbors equally — unnatural, because a pixel 2 hops away shouldn't matter as much as an adjacent one. Gaussian blur uses the 2D Gaussian function: the center pixel gets the highest weight, dropping smoothly with distance. The result looks natural, no box-shaped artifacts.


        
⚠️
Kernel size must be odd — OpenCV requires (3,3), (5,5), (7,7)... Pass (0,0) to let OpenCV auto-compute size from sigma. Rule of thumb: ksize ≈ 6σ + 1.

Median Filter
Salt-and-pepper noise removal — sort the neighborhood, pick the middle

        

        
Gaussian blur averages its neighborhood — a salt spike at 255 bleeds into surrounding pixels. The median filter is non-linear: it sorts all neighborhood values and returns the middle one. An outlier of 255 ends up last in a sorted list of ~80s and never reaches the output.


        
Salt-and-pepper noise: random pixels set to 0 (pepper) or 255 (salt). Common from cheap sensors, transmission errors, old scans. Gaussian spreads the outlier; median eliminates it entirely.

How median filter works — step by step

          

            
1
3×3 neighborhood with a salt spike (255) at center
82  90  88  /  85  255  91  /  79  87  83

            
2
Sort all 9 values ascending
79 · 82 · 83 · 85 · 87 · 88 · 90 · 91 · 255

            
3
Pick the middle value (index 4 of 9) — spike is gone
Output = 87  noise eliminated without blurring neighbors

Unsharp Masking
Sharpen by adding back the high-frequency detail you subtracted

        

        
Despite the name, unsharp masking sharpens. Blur the image → subtract it from the original → what remains is the detail (edges, texture) that blur removed. Add that back, amplified. The "mask" is the blurred version used to isolate what to enhance.


        

          
Unsharp Mask Formula

          

            output
            =
            original
            +
            amount
            ×
            (original
            −
            blurred)
          

          

            blurred = GaussianBlur(original)
            amount = sharpening strength (0.5–2.0)
            (original − blurred) = the detail mask

unsharp masking


# Manual (most readable)
blur   = cv2.GaussianBlur(img, (0,0), sigmaX=2)
mask   = img.astype(np.float32) - blur
amount = 1.5
sharp  = np.clip(img + amount * mask,
                 0, 255).astype(np.uint8)

# addWeighted shortcut:
sharp = cv2.addWeighted(
    img,  1 + amount,
    blur, -amount, 0
)

          

            
 Amount vs Radius guide

            
    
              
  • Small σ + high amount → crisp edges, but adds texture noise
    • 
              
  • Large σ + low amount → wider contrast boost, fewer artifacts
    • 
              
  • amount > 2.0 → halos appear around edges
    • 
              
  • Typical: σ = 1–2, amount = 0.5–1.5
    •

Otsu's Adaptive Binarization
Let the histogram find the optimal threshold automatically

        

        
Binary thresholding turns grayscale → black/white: pixels above threshold T become 255, below become 0. The problem: which T? A fixed T breaks under different lighting. Otsu's method scans all 256 T values, computes how well each separates the histogram into two clusters, and picks the T that minimizes intra-class variance.


        
Otsu works best on bimodal histograms — two clear peaks (dark background + bright foreground). For uneven lighting, use cv2.adaptiveThreshold() which computes a local T per region.

Otsu's algorithm — three steps

          

            
1
Build normalized histogram: p[i] = fraction of pixels at intensity i
hist, _ = np.histogram(gray.ravel(), 256, [0,256])
p = hist / hist.sum()   # sum = 1.0

            
2
For each t, compute between-class variance (higher = better separation)
w0 = p[:t].sum()     w1 = 1 − w0
μ0 = (i·p[:t]).sum()/w0     μ1 = (i·p[t:]).sum()/w1
σ²_between = w0 · w1 · (μ0 − μ1)²

            
3
Pick t* = argmax σ²_between — OpenCV does this in one call
t, binary = cv2.threshold(gray, 0, 255,
   cv2.THRESH_BINARY + cv2.THRESH_OTSU)

          

        


        

          
cv2 · Otsu threshold


gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# Fixed threshold
_, b_fixed = cv2.threshold(
    gray, 127, 255, cv2.THRESH_BINARY)

# Otsu: T=0 tells OpenCV to auto-find it
t_otsu, b_otsu = cv2.threshold(
    gray, 0, 255,
    cv2.THRESH_BINARY + cv2.THRESH_OTSU)
print(f"Otsu T = {t_otsu}")

# Blur first for noisy images
blr = cv2.GaussianBlur(gray, (5,5), 0)
t2, b2 = cv2.threshold(blr, 0, 255,
    cv2.THRESH_BINARY + cv2.THRESH_OTSU)

          
Adaptive threshold (uneven lighting)


# When Otsu fails (shadows, gradients)
adaptive = cv2.adaptiveThreshold(
    gray,
    255,
    cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
    cv2.THRESH_BINARY,
    blockSize=11,  # local region (odd)
    C=2            # constant subtracted
)
# blockSize: how local the threshold is
# C=2 avoids noise becoming foreground
