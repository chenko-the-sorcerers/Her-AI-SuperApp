Mengapa MLP Gagal pada Gambar?

Dua masalah fundamental yang membuat MLP tidak cocok untuk computer vision

Sebelum CNN, orang mencoba menggunakan Multilayer Perceptron (MLP) untuk klasifikasi gambar. Hasilnya buruk — bukan karena MLP bodoh, tapi karena gambar punya properti unik yang tidak bisa ditangani MLP. Ada dua masalah mendasar: ledakan parameter dan hilangnya konteks spasial.
Problem 1
Parameter Explosion
Gambar 224×224×3 = 150,528 nilai. Satu hidden layer 512 neuron → butuh 77 juta parameter. Tidak scalable, mudah overfit.

Spatial Context Lost

Problem 2 — Hilangnya Konteks Spasial

Flatten menghancurkan informasi "siapa bersebelahan dengan siapa"

Bahkan jika kita abaikan masalah parameter, ada masalah yang lebih mendasar: ketika kita flatten gambar menjadi vektor 1D, kita menghancurkan semua informasi tentang posisi dan hubungan antar piksel.

3
Di CNN: hanya piksel dalam receptive field yang saling berpengaruh
// Filter 3×3 di posisi (r,c) hanya menggunakan:
I[r,c], I[r,c+1], I[r,c+2]
I[r+1,c], I[r+1,c+1], I[r+1,c+2]
I[r+2,c], I[r+2,c+1], I[r+2,c+2]
// Piksel jauh tidak mempengaruhi output lokal — sesuai kenyataan!

Inspirasi Biologis — Korteks Visual

CNN bukan diciptakan dari nol — terinspirasi cara otak manusia melihat

Pada 1959, Hubel & Wiesel menemukan bahwa neuron di korteks visual kucing merespons secara spesifik terhadap pola tertentu — bukan semua piksel sekaligus. Simple cells merespons orientasi tepi tertentu. Complex cells merespons tepi dari berbagai posisi. Inilah yang menginspirasi CNN.

Kalkulasi Receptive Field
Seberapa besar area input yang dilihat tiap neuron?
Receptive field adalah area di gambar asli yang mempengaruhi aktivasi satu neuron. Semakin dalam layer, semakin besar receptive field-nya.
1
Layer 1: Conv 3×3, stride 1
RF1 = 3×3 area pada gambar input
2
Layer 2: Conv 3×3 lagi, stride 1
// Layer 2 melihat 3×3 area di output Layer 1
// Tiap sel Layer 1 sudah mewakili 3×3 area asli
RF2 = 3 + 2 (tambah 1 di tiap sisi) = 5×5
3
Formula umum: L layer Conv 3×3 tanpa pooling
RFL = 1 + L × (k−1)
// k = ukuran kernel (3), L = jumlah layer
5 layer Conv 3×3 → RF = 1 + 5×(3−1) = 11×11 piksel
4
Dengan MaxPool 2×2 (stride 2) setelah tiap 2 Conv
// Pooling melipatgandakan effective receptive field
Setelah 2 Conv + 1 Pool: RF × 2 = 10×10
VGG-16 akhir: RF ≈ 196×196 dari gambar 224×224
Persis seperti simple cells di V1 melihat patch kecil, complex cells di V4 melihat area lebih besar — receptive field CNN secara alami tumbuh seiring kedalaman, membentuk hirarki representasi yang sama dengan korteks visual manusia.

Solusi CNN — Tiga Properti Kunci

Bagaimana CNN mengatasi semua masalah MLP sekaligus

CNN mengatasi kedua masalah MLP dengan tiga properti fundamental yang bekerja bersama: local connectivity, parameter sharing, dan equivariance.
1
Local Connectivity
Tiap neuron hanya terhubung ke area kecil (receptive field) dari layer sebelumnya, bukan ke semua piksel.
neuron(r,c) = f( I[r:r+k, c:c+k] )
k = ukuran kernel (misal 3)
✓ Menangkap struktur spasial lokal
2
Parameter Sharing
Satu set weight (filter) digunakan di semua posisi gambar. Filter tepi horizontal yang sama bekerja di seluruh bagian gambar.
W(r,c) = W(r′,c′) = K ∀ (r,c),(r′,c′)
Satu kernel K berlaku universal di seluruh gambar
✓ Efisiensi parameter: 3×3=9, bukan 9×W×H
3
Translation Equivariance
Jika objek bergeser di input, feature map-nya bergeser dengan cara yang sama — tidak perlu belajar ulang.
Conv(TΔ(I)) = TΔ(Conv(I))
TΔ = geser sebesar Δ piksel
✓ Robust terhadap posisi objek
⚡ Perbandingan Akhir
